# Практична робота №11–12

## Тема
Взаємодія Front-end із сервером. REST API та CRUD-операції засобами JavaScript

## Опис роботи
У даній практичній роботі реалізовано веб-застосунок у вигляді каталогу, який взаємодіє з REST API за допомогою fetch() та async/await.  
Джерелом даних виступає локальний сервер json-server.

Проєкт демонструє повний цикл роботи з даними: отримання, створення, редагування та видалення записів із синхронізацією інтерфейсу.

## Реалізований функціонал

### CRUD-операції
- GET — отримання списку елементів і відображення каталогу
- GET by ID — отримання даних одного елемента для редагування
- POST — створення нового елемента через форму
- PATCH — редагування існуючого елемента
- DELETE — видалення елемента з підтвердженням

### Робота з інтерфейсом
- Динамічний рендеринг каталогу (картки)
- Модальне вікно з деталями елемента
- Повідомлення про успішні дії та помилки (UI замість alert)
- Стан завантаження (loading)
- Обробка помилок (error)
- Повідомлення при відсутності результатів (empty state)

### Функції каталогу
- Пошук елементів
- Фільтрація за категоріями
- Сортування за ціною
- Поступове завантаження (load more)

### Форма
- Створення та редагування в одній формі
- Автоматичне заповнення даних при редагуванні
- Клієнтська валідація полів

### API-шар (api.js)
Реалізовано окремі функції для роботи з сервером:
- getItems()
- getItemById(id)
- createItem(data)
- updateItem(id, data)
- deleteItem(id)

## Використані технології
- HTML5
- CSS3
- JavaScript (ES6+)
- fetch API
- async/await
- json-server

## Запуск проєкту

1. Встановити json-server:
```
npm install -g json-server
```

2. Запустити сервер:
```
json-server --watch db.json --port 3000
```

3. Відкрити файл:
```
index.html
```
